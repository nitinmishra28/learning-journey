# 🧩 Important Problem-Solving Patterns

Learning patterns is more useful than memorizing individual solutions.

Important patterns include:

### Two Pointers

Used when working with arrays, strings, and linked lists using two positions.

### Sliding Window

Used when dealing with continuous subarrays or substrings.

### Fast and Slow Pointers

Commonly used in Linked Lists for cycle detection and finding the middle.

### Binary Search

Used to reduce the search space by half.

### Binary Search on Answer

Used when the answer lies within a range and we can check whether a value is possible.

### Prefix Sum

Used for efficient range calculations.

### Hashing

Used for fast lookup, frequency counting, and duplicate detection.

### Monotonic Stack

Used for Next Greater, Next Smaller, and similar problems.

### Merge Intervals

Used when working with overlapping ranges.

### Recursion and Backtracking

Used when exploring multiple choices.

### BFS

Used for level-wise traversal and shortest paths in unweighted graphs.

### DFS

Used for deep traversal, connected components, and recursive exploration.

### Heap / Top K

Used when finding the largest, smallest, or top K elements.

### Greedy

Used when making locally optimal decisions leads to the global solution.

### Dynamic Programming

Used when recursive subproblems repeat.

# 🔍 Problem-Solving Approach

Whenever solving a new DSA problem, follow this process:

```text
1. Understand the problem
        ↓
2. Identify input and output
        ↓
3. Check constraints
        ↓
4. Think about the brute-force solution
        ↓
5. Analyze brute-force complexity
        ↓
6. Identify the pattern
        ↓
7. Think about optimization
        ↓
8. Dry run the solution
        ↓
9. Write the code
        ↓
10. Test edge cases
        ↓
11. Analyze time and space complexity
```

---

# 🧠 How to Identify a Pattern

Instead of immediately thinking about code, ask questions like:

```text
Is the input sorted?
        → Think Binary Search or Two Pointers

Is the problem asking about a continuous range?
        → Think Sliding Window or Prefix Sum

Do I need fast lookup?
        → Think Hash Map or Hash Set

Am I repeatedly finding minimum or maximum?
        → Think Heap

Is it a Linked List cycle or middle problem?
        → Think Slow and Fast Pointers

Do I need the next greater or smaller element?
        → Think Monotonic Stack

Do I need to explore every possible choice?
        → Think Recursion or Backtracking

Are the same recursive states repeating?
        → Think Dynamic Programming

Is the problem asking about shortest distance?
        → Think BFS or Shortest Path Algorithms
```
